# Student Registration System (Clean Architecture)

## Опис проекту
Цей проект демонструє реалізацію реєстрації студента на курс із використанням принципів чистої архітектури.

### Реалізований Use Case
**Реєстрація студента на навчальний курс із перевіркою доступних місць.**
Система перевіряє, чи є вільні місця на курсі, перш ніж інкрементувати кількість зареєстрованих учасників.

### Обрана архітектура
Використано **Clean Architecture (Onion Architecture)**.
**Чому її обрано:**
- Незалежність від фреймворків
- Тестованість
- Гнучкість

### Розподіл відповідальностей
- **Domain (Entities):** містить об'єкт `Course` та метод `hasAvailableSlots`. Це чиста бізнес-логіка.
- **Application (Use Cases):** координує потік даних. Викликає репозиторій та методи домену.
- **Infrastructure (Adapters):** реалізація `TypeORMCourseRepository`. Тут знаходяться технічні деталі роботи з БД.
- **Interface (Controllers):** приймає HTTP-запити та передає дані в Use Case.

### Порти та адаптери
- **Порт:** `ICourseRepository` (абстрактний клас/інтерфейс).
- **Адаптер:** `TypeORMCourseRepository` (конкретна реалізація для SQLite/PostgreSQL).

## Як запустити:
1. встановіть залежності: `npm install`.
2. налаштуйте підключення до БД у файлі конфігурації TypeORM.
3. запустіть додаток: `node src/index.js`.
